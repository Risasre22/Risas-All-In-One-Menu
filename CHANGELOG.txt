RISA'S ALL IN ONE MENU - CHANGELOG
===================================
Newest first. "In progress" = built/deployed locally but not yet released.
Claude keeps this file up to date automatically as changes are made.


1.5.5
-----
New:
- Added support for Mod Function Menu
- Added Language support: New dropdown so you can pick which translation Risa loads (changes apply right away, note that you need the translation.json in "SKSE\Plugins\RisaAllInOneMenu\Translations\" folder for it to show up more info on the mod page)
-Added a Font Scale slider: A few elements still scale along with the text by design, because scaling font alone without adjusting everything around it just breaks the layout however this will help alot for people who use different fonts.

Changed:
- FLICK: API access to Keys/Menu no more .ini edits, thanks to the author Fuzzlesz adding support~
- Dragonborn's Toolkit: API access to Keys/Menu no more .ini edits, thanks to the author ModsLab!~
- Key tooltips now show exactly which key a mod is sitting on, or if it's handled through an API.
- Overhauled a bunch of tooltips to be simpler and less confusing.
- New tab colors so hover, selected, and unselected tabs are easy to tell apart.
- Big pass on how the menu scales, etter spacing and readability.
Fixed:
- The Advanced Bindings popup no longer disappears when you pick a launcher key.
- SearchUI now closes with your menu key and Esc.
- Debug Menu now closes with Esc.
- many, many elements being misplaced when scaling at extreme sizes.

1.5.4
-----
New:
- Added a Support for SearchUI.
- Added Support for Quick Armor Rebalance
- Added a Mod Configuration Menu (MCM) button. You can pick its hotkey in Settings.
- Added an Exclusions tab: turn off my management for most mods so it keeps its own hotkey.
- SKSE Menu Framework is now handled through its own built-in switch instead of editing its settings file.
- Every setting row now has a small keyboard button that opens the Advanced Bindings popup (modifiers, double-tap, hold, and reset to default). GREEN means its not default.
- The menu title now has an info circle - click it for a full help tooltip (needs work still).
- Key-binding boxes now give up after 3 seconds if you don't press anything, so you can't get stuck waiting.
- Changed the boxes to toggles it looks better
- Added a "Keep the game running while the menu is open" option (Settings, off by default).
- Added a "Let the menu open while the console is open" option (Settings, off by default).

Fixed:
- dMenu NG (Risa Version) now the only one supported.
- ReShade's overlay key now goes back to Home properly after a restore.
- SearchUI: removed my Esc/F1 close for it, since SearchUI already closes with its own built-in Tab key (will look into it more later to be able to close with set menu key and esc i legitmately couldnt get it to work)
- While the console is open, menus that can't be used from it (IED, Debug Menu, Improved Camera, KreatE, Skyrim Party Sheet, CatMenu, Dragonborn's Toolkit) are greyed out.
- Fixed the Debug Menu button not opening the menu at all (only closes on menu key not ESC yet)
- "Restore for uninstall" no longer wrongly says "incomplete" when a mod's config simply didn't have a key it manages.
- The MCM button now shows at the main menu too (greyed until a save loads); before, it only appeared after a save was loaded.
- In Settings, excluded mods now only grey out their hotkey controls - you can still switch the mod's name on and off.
- Settings and Exclusions lists now work fully with controller D-pad and keyboard navigation.
- Exclusions tab: the hotkey column now shows your current key when a mod is managed, or its own native key when it's excluded.
- Cleaned up and simplified all the settings descriptions and tooltips so they're easier to read on screen.


Internal:
- Rebuilt against the actively-maintained CommonLibSSE-NG fork (alandtse/CommonLibVR, ng branch) for newer game-version support and fixes. SE/AE only for now (VR maybe later).
