RISA'S ALL IN ONE MENU - INTEGRATION AND HIDDEN KEY LEDGER
Last updated: 2026-07-01


PURPOSE
-------

This file answers two practical questions:

1. Which integrations still depend on simulated or parked hotkeys?
2. Which F13-F24 keys are safe to reuse?

For the complete API, maintenance, and risk matrix, see INTEGRATION_STATUS.md.


STATUS DEFINITIONS
------------------

[FREE]      Safe to assign to another integration.
[RESERVED]  A supported mod still listens for or receives this key.
[FALLBACK]  Reserved for a compatibility path, even if unused on this PC.

Integration methods:

[API]       Official interface supplied by the target mod or framework.
[COMMAND]   Stable command supplied by the target mod.
[INTERNAL]  Verified private function/state integration. Keyless, but version-sensitive.
[SINK]      Delivers input only to the target mod's own SKSE input sink; no key is reserved.
[HOTKEY]    The launcher still invokes a relocated or native hotkey.

DIRECT, BUT A KEY REMAINS RESERVED
----------------------------------

Open Animation Replacer
  Method: verified internal UIManager state.
  F13 remains reserved because OAR's native listener is still parked there.
  Never set OAR's key to 0: OAR interprets scan code 0 as the left mouse button.

dMenu NG
  Method: dMenu NG external-control API (SKSE message 0x444D4946 "DMIF" from sender "dmenu", with
    a dMenu_GetInterface() export fallback). Open / close / toggle / is-open go straight through
    the API -- no key simulation and a real menu-state query, so no more open/close desync.
  F14 remains reserved because dMenu's native listener is still parked there to free Home.
  Stock dMenu (no API) falls back to the F14 engine-injection path automatically (HasDMenuApi gate).

MODIFIER CHORDS
---------------

Ctrl, Shift, and Alt do not automatically multiply the available key pool. Some mods ignore
modifiers and react to the base key. A chord is reusable only after that listener is tested or
isolated. Current hidden-key modifier allocations: none.


MAINTENANCE RULES
-----------------

1. Update this file whenever a key, fallback, listener, or opening method changes.
2. Mark a key FREE only when no supported mod can react to its physical or injected event.
3. Record author outreach and update activity in INTEGRATION_STATUS.md.
4. Unknown DLL versions must fall back safely instead of calling unverified internal code.
