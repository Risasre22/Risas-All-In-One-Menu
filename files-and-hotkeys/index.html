    Files & Hotkeys Edited¶

    “Original” means the value my menu captures before managing a mod. Restore for Uninstall returns captured values whenever possible; the listed upstream defaults are used only as fallbacks.

    Keys Freed for You — 26 bindings using 19 distinct hotkeys

    If every supported integration is installed, my menu manages 26 default hotkey bindings across 20 mods. These bindings use 19 distinct hotkeys, because several mods share F1, Home, F6, or End. MCM and Quick Armor Rebalance are also supported, but neither has a native default hotkey.

    • SKSE Menu Framework: F1
    • dMenu NG: Home
    • FLICK: F7
    • CatMenu: F6
    • Dragonborn’s Toolkit: F1
    • Debug Menu: F1
    • SearchUI: F4
    • Mod Function Menu: F1
    • Modex: Delete
    • Hotkey Reminder: F11
    • Open Animation Replacer: Shift + O
    • Immersive Equipment Displays: Backspace
    • Outfit Preview Selector: Numpad 1
    • Skyrim Vanity System: F6
    • Skyrim Party Sheet: X (Settings), F6 (Party Sheet), Y (Inspect Card), U (Character Sheet) — 4 bindings
    • ENB Editor: Shift + Enter
    • Community Shaders: End (Main), Shift + End (Editor), F10 (Overlay), Numpad * (Effects) — 4 bindings
    • Improved Camera: Shift + Home
    • ReShade: Home
    • KreatE: End

    Files my menu edits¶

    Only the listed hotkey values are changed. Other settings in each file are preserved.

    Open Animation Replacer
    Path: SKSE\Plugins\OpenAnimationReplacer.ini
    Default key: Shift + O
    Default values: uToggleUIKey = 24 | uToggleUIKeyShift = 1 | uToggleUIKeyCtrl = 0 | uToggleUIKeyAlt = 0
    Changed to: uToggleUIKey = F13 | Shift/Ctrl/Alt = 0. F13 is used as an unpressable parking key.

    Improved Camera SE (the NG/API path may require no file edit)
    Path: SKSE\Plugins\ImprovedCameraSE\ImprovedCameraSE.ini
    Default key: Shift + Home
    Default value: MenuKey = 0x24 (Home)
    Changed to: MenuKey is normalized to Home when needed. My menu then hides the chord only from Improved Camera, leaving Shift + Home available elsewhere.

    Immersive Equipment Displays
    Path: SKSE\Plugins\ImmersiveEquipmentDisplays.ini
    Default key: Backspace
    Default values: ToggleKeys = 0x0E | OverrideToggleKeys = false
    Changed to: ToggleKeys = F16 | OverrideToggleKeys = true. F16 is used as an unpressable parking key.

    Debug Menu
    Path: MCM\Settings\DebugMenu.ini (or MCM\Config\DebugMenu\settings.ini)
    Default key: F1
    Default value: uOpenMenuHotkey = 59
    Changed to: uOpenMenuHotkey = F17 only when it conflicts with the launcher or contains an older relocation made by my menu. Otherwise its configured key is left in the file and intercepted live.

    KreatE
    Path: KreatE\UserSettings.ini
    Default key: End
    Default value: GUIToggleKeys = 0x23
    Changed to: GUIToggleKeys = F18 (unpressable parking key).

    Community Shaders
    Paths: SKSE\Plugins\CommunityShaders\SettingsUser.json and SettingsDefault.json when present
    Default keys: End / Shift + End / F10 / Numpad *
    Default values under “Menu”: "ToggleKey": 35 | "CSEditorToggleKey": [16, 35] | "OverlayToggleKey": 121 | "EffectToggleKey": 106
    Changed to: ToggleKey = F19 | CSEditorToggleKey = F15 with no modifier | OverlayToggleKey = F23 | EffectToggleKey = F24.

    Skyrim Vanity System
    Path: SKSE\Plugins\SkyrimVanitySystem\settings.json
    Default key: F6
    Default value: "toggleKey" = 64
    Changed to: "toggleKey" = 109 (F22 unpressable parking key). My menu injects F22 to open or close the menu and restores the captured key on exclusion or uninstall.

    Modex
    Path: Interface\Modex\user\settings.json
    Default key: Delete
    Default value: "Open Menu Keybind" = 211
    Changed to: "Open Menu Keybind" = 0 (disabled). My menu refreshes the setting through Modex's official native API and opens Modex directly. Closing uses a UI-message workaround because Modex 3.0's CloseMenu API currently reopens the menu.

    Dragonborn’s Toolkit legacy fallback (v0.5+ uses its live API and does not need this edit)
    Path: SKSE\Plugins\SkyrimCheatMenu.json
    Default key: F1
    Default value: "toggleKey": "F1"
    Changed to: "toggleKey": 0 only when the runtime hotkey API is unavailable.

    ReShade
    Path: ReShade.ini in the game folder
    Default key: Home
    Default value: KeyOverlay=36,0,0,0
    Changed to: KeyOverlay=0,0,0,0. The file disables the native key; my ReShade add-on opens and closes the overlay through the API.

    Managed live without editing another mod’s file¶

    • SKSE Menu Framework: native F1 listener disabled/restored through its API.
    • dMenu NG: Home disabled/restored through the v2 API. Stock dMenu without that API is left unmanaged.
    • Photo Mode: opened and closed through po3_photomode.TogglePhotoMode(true/false). Its primary open key is disabled by default; no settings file is edited.
    • FLICK: F7 disabled/restored through its hotkey API.
    • SearchUI: F4 parked live on F18 through Papyrus.
    • Outfit Preview Selector: Numpad 1 parked live on F18 through its Papyrus hotkey property; opening uses its Papyrus method and closing uses the OPS_CloseMenu mod event; no INI is edited.
    • Hotkey Reminder: F11 listener disabled/restored through ToggleHotkey(). Its settings INI is read only.
    • Skyrim Party Sheet: X, F6, Y, and U are intercepted live; PartyUserSettings.ini is read only.
    • ENB Editor: Shift + Enter is hidden only from ENB through caller-scoped input handling.
    • Dragonborn’s Toolkit v0.5+: F1 listener disabled/restored through its runtime API.
    • Mod Configuration Menu: no native hotkey to edit.
    • Quick Armor Rebalance: no native default hotkey to edit.

    My menu’s support files¶

    These files belong to my menu. They are not files from another supported mod.

    Settings and backups¶

    File Purpose
    SKSE\Plugins\RisaAllInOneMenu.ini Stores my menu settings and original-hotkey options.
    SKSE\Plugins\RisaAllInOneMenu_OriginalHotkeys.json Stores captured settings used by Restore User Original Settings.
    SKSE\Plugins\RisaAllInOneMenu_OriginalHotkeys_Completed.json Archives a completed original-settings backup after a successful uninstall restore instead of deleting it.
    SKSE\Plugins\RisaAllInOneMenu\Translations\ Contains the language JSON files shown in the language dropdown.
